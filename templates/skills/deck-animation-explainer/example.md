# AnySearch AI Skill：搜索不是聊天框

## Global Style

- audience: AI agent / coding agent 使用者
- tone: technical educational, concise, video-friendly
- aspect: 16:9
- target: 6 slide animation explainer

## Slides

### 01 Hook

- headline: 搜索不是聊天框，是 Agent 的外部感知器
- body: AnySearch 把“去网上找答案”变成可约束、可复用、可审计的工具调用。
- visual: pulsing sensor ring connected to a small agent node
- motion: sensor ring expands slowly, then locks on a search beam
- speaker_note: 先建立反常识：搜索不是万能问答，而是工具接口。

### 02 Concept

- headline: 它给 Agent 四种搜索能力
- body: general search、vertical search、batch search、extraction。不同任务选择不同入口。
- visual: four capability tiles orbit around a central tool node
- motion: tiles enter one by one
- speaker_note: 强调能力分层，不要把它当一个普通搜索框。

### 03 Pipeline

- headline: 正确流程是先定任务，再选目录
- body: 识别任务 → 读取 domain catalog → 构造 query → 抽取网页 → 合并上下文。
- visual: five-step pipeline with arrows and one highlighted current step
- motion: arrow line draws from left to right
- speaker_note: 这页说明 Agent 应该如何调用工具。

### 04 Compare

- headline: 错误用法是猜参数，正确用法是先看边界
- body: 猜测搜索参数会造成噪声；目录优先能让查询更稳定。
- visual: left column "guess params" with noisy dots, right column "catalog first" with clean route
- motion: noisy dots fade down, clean route brightens
- speaker_note: 对比错误和正确的使用姿势。

### 05 Stack

- headline: 它是外部上下文入口，不是最终判断者
- body: AnySearch 负责带回材料；Agent 仍要做筛选、归纳、引用和决策。
- visual: three-layer stack: Search Source, Extracted Evidence, Agent Reasoning
- motion: layers stack upward
- speaker_note: 明确系统角色，避免过度神化工具。

### 06 Closing

- headline: 好的搜索工具让 Agent 更可控
- body: 先收窄任务，再选择搜索面，最后保留证据链。
- visual: final route from question to evidence to answer
- motion: route completes and progress bar reaches 100%
- speaker_note: 结尾给可复用方法。
