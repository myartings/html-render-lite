AI slides 不是填模板,
是现场设计视觉系统。

先从 brief 里判断受众、语气和场景, 再手写一套配色、字体、背景和版式语法。下面 6 张把这个工作流讲完。

01 / 07

四个决策 先于写代码。

audience

谁会看

工程师、投资人、学生和运营团队需要不同的信息密度。

tone

说话方式

工程感、编辑感、学术感或品牌感, 必须落到 CSS 选择。

system

视觉系统

token、背景、字体和页面类型组成一套小型 design system。

motion

演示节奏

键盘翻页、错峰 reveal 和页码提示服务讲述, 不是装饰。

02 / 07
scenario 01 · 工程师向

工程感不是代码块,
是约束和节奏。

brief 暗示

  • 面向开发者或 AI 工具用户
  • 内容是工作流、runtime、agent、infra
  • 需要短句和清晰结构

设计落点

  • 深底 + 暖色 accent + mono metadata
  • 1px grid 和 radial glow 做技术空间
  • scenario 用左右对照提高扫读速度
03 / 07
scenario 02 · 避开套路

不要让模型退回
紫色渐变 SaaS。

常见漂移

  • 白底大圆角卡片堆叠
  • Inter/Roboto + generic dashboard
  • emoji、玻璃拟态、无意义 bento

修正方式

  • 明确禁止默认 AI 视觉套路
  • 要求 token、背景和版式都服务内容
  • 每页只保留一个 takeaway
04 / 07
scenario 03 · 页面类型

好看的 deck 来自
少量版式复用。

cover

一句话定调

用最大标题和一个视觉 hack 给观众记忆点。

concept

概念网格

最多四个概念, 每个只解释一层。

scenario

左右对照

用户动作与系统产出分列, 节奏稳定。

action

收尾行动

最后一页给下一步, 不写 Thank you。

05 / 07
scenario 04 · 代码实现

几十行 JS 就够,
不要引 slide 框架。

必须保留

  • Arrow / Space 翻页
  • Home / End 跳转
  • 移动端纵向阅读

可以加分

  • click 左右区域翻页
  • hash 同步和 reduced motion
  • 简洁 pager 与键盘 hint
